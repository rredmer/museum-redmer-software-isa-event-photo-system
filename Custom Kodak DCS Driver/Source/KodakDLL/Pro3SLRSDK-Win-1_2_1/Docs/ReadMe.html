5. New Features (Since version 1.1.3)

  • Camera control and image processing capability for DCS 760M
  • New API function KPDCGetImageRectangleWithTest(). This function has more robust parameter error checking than the original KPDCGetImageRectangle(). Though KPDCGetImageRectangle() is still supported, it is recommended for developers to begin to use this new implementation.

7. Known Issues

  1. On Windows 98SE and ME communications with the camera or back may be lost if two cards are inserted The scenario where this happens the most is as follows:
    • Insert two PCMCIA cards in the camera or back
    • Format each card using the menu options of the camera or back
    • Shoot a couple images to card 0
    • Shoot a couple of images to card 1
    • Events for the card 0 shoot will be handled correctly but the events for shooting to card 1 get lost causing the SDK to become unresponsive. There is no work around at this time.
  2. Some Windows 98SE and ME systems fail to find a camera or back. The device does appear in the Device Manager but communications through the SDK to the camera fail. Windows 2000 and XP Professional are the recommended operating systems for tethered operation.

8. Functional Clarifications

  1. For maximum performance in tethered mode operation you should use the AC Adapter as the power source
  2. For tethered support on Macintosh systems you need to drag the files in the MacExtensions folder to your system extensions folder. The drivers included with this version are different than what are included with the DCS SDK.
  3. For tethered support on Windows system plug in your camera and when the "New Device Found" dialog opens follow the instructions on loading the appropriate driver. Point to the driver for the operating system you are using. The drivers included with this version are different than what are included with the DCS SDK.

10. Service and Support

If you suspect a bug in the SDK you downloaded, please first check Kodak's web page to ensure that you have the latest version and the camera or back has the latest version of firmware. To report a problem, request a feature, or request clarification of a function, please complete the issue report form found in the Developer Relations area of the Kodak DRG Web site. The URL is http://www.kodak.com/go/drg.

After submitting the report you will receive a response indicating that we have received your report. At the same time your report will be fed into our internal support database that will begin our process of handling the report. The priority assigned to your report will depend on the severity of the problem. Fatal issues will receive the highest priority. Please indicate platform and host configuration such as memory, processor, speed, and your hardware settings, including SCSI and IEEE 1394 adapters, when reporting an issue. All issues will be reviewed by our Change Control Board to determine the length of time it will take to implement and test a code change to repair the bug. You will be notified of the expected delivery date and version number of the library that will have the bug repaired. If the Change Control Board does not fully understand the issue report you will be contacted for more information.

Requests for added features will receive the same treatment but with a lower priority than reported bugs.

A question about a particular function of the SDK will not initiate a review by our Change Control Board but you will receive an answer from our development community.
